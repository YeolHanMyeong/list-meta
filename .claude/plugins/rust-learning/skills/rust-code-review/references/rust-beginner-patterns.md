# Rust 초보자 패턴 및 안티패턴

## 좋은 패턴 (권장)

### 에러 처리
- `?` 연산자로 에러 전파 → `unwrap()` 남발보다 훨씬 좋음
- `main()`의 반환 타입을 `Result<(), Box<dyn std::error::Error>>`로 설정
- 사용자 친화적 에러 메시지: `eprintln!`로 stderr에 출력

### 소유권과 빌림
- 문자열을 받을 때 `&str`로 빌려쓰기 (소유권 이전 불필요 시)
- `.clone()`은 필요할 때만 — 무분별한 clone은 안티패턴
- 함수 인자로 `&Path` 사용 (`PathBuf` 대신, 읽기만 할 때)

### 이터레이터
- `for` 루프보다 이터레이터 체인이 관용적인 경우가 많음
- `.filter().map().collect()` 패턴
- `collect::<Vec<_>>()` — 타입 추론이 안 될 때 터보피시 사용

### 구조체 설계
- 관련 데이터를 구조체로 묶기
- `#[derive(Debug)]` — 디버깅 출력용
- 필드가 없을 수 있으면 `Option<T>` 사용

### 모듈 구성
- 파일이 길어지면 모듈로 분리
- `mod meta;` → `src/meta.rs` 또는 `src/meta/mod.rs`
- `pub` 키워드로 공개 범위 제어

## 안티패턴 (주의)

### unwrap() 남발
- **문제**: 예상치 못한 에러에서 패닉 발생
- **대안**: `?`, `unwrap_or_default()`, `match`, `if let`
- **예외**: 테스트 코드나 절대 실패하지 않는 경우는 OK

### 불필요한 .clone()
- **문제**: 불필요한 메모리 할당
- **힌트**: 참조(`&`)로 충분한지 먼저 확인

### 문자열 타입 혼란
- `String` — 소유한 문자열, 힙 할당
- `&str` — 빌린 문자열 슬라이스
- **규칙**: 소유가 필요하면 `String`, 읽기만 하면 `&str`

### 거대한 main()
- **문제**: 모든 로직이 `main()`에 집중
- **대안**: 기능별 함수로 분리 → 가독성, 테스트 용이성 향상

### 하드코딩된 값
- **문제**: 매직 넘버, 매직 스트링
- **대안**: 상수(`const`), 설정 구조체, CLI 옵션

### 미사용 import/변수
- **문제**: `#[warn(unused_imports)]` 경고 무시
- **대안**: `cargo clippy`로 정기 점검, 경고를 0으로 유지

## Cargo 도구 활용

### cargo clippy
- Rust 린터 — 일반적인 실수와 개선 포인트를 잡아줌
- `cargo clippy -- -W clippy::pedantic` — 더 엄격한 검사

### cargo fmt
- 코드 포맷터 — 일관된 스타일 유지
- `rustfmt.toml`로 설정 커스터마이즈 가능

### cargo check
- `cargo build`보다 빠른 타입 검사
- 코드 수정 후 빠르게 에러 확인할 때 유용
