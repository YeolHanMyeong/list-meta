# Rust 코드 리뷰 스킬

## 핵심 원칙

1. **코드를 대신 수정하지 않는다.** 문제점을 지적하고 방향만 제시한다.
2. **긍정적 피드백을 먼저 한다.** 잘한 부분을 구체적으로 칭찬한다.
3. **최대 5개 개선 제안.** 너무 많은 피드백은 학습을 방해한다.
4. **우선순위를 명시한다.** 중요한 것부터 순서대로 제안한다.

## 리뷰 절차

### 1단계: 코드 읽기
- `src/main.rs`와 관련 소스 파일들을 읽는다.
- `Cargo.toml`에서 의존성 상태를 확인한다.

### 2단계: 긍정적 피드백 (2~3개)
다음 중 해당되는 것을 칭찬한다:
- 올바른 에러 처리 (`Result`, `?` 연산자)
- 적절한 타입 사용
- 깔끔한 코드 구조
- 관용적인 Rust 패턴 사용
- 이전 단계 대비 개선된 점

### 3단계: 개선 제안 (최대 5개)
각 제안에 포함할 내용:
- **무엇이** 개선 가능한지 (구체적 위치)
- **왜** 개선해야 하는지 (이유)
- **어떤 방향으로** 개선할 수 있는지 (힌트, 키워드)
- 절대 완성된 코드를 제공하지 않는다

### 4단계: 현재 단계 기준 평가
- `references/rust-beginner-patterns.md`의 패턴/안티패턴 참조
- 현재 학습 단계의 체크리스트 기준으로 완성도 평가

## 응답 형식

```
## 잘한 점 👍
- (구체적 칭찬 1)
- (구체적 칭찬 2)

## 개선 제안
1. [중요도: 높음] (제안 내용 + 힌트)
2. [중요도: 중간] (제안 내용 + 힌트)
...

## 현재 진행도
Stage N 체크리스트: X/Y 완료
```
