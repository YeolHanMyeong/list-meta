# /hint 커맨드

현재 구현 상태를 분석하고, 사용자가 막힌 부분에 대해 점진적 힌트를 제공한다.

## 실행 절차

1. **현재 상태 파악**:
   - `src/main.rs` 및 `src/` 하위 파일들을 읽는다.
   - `Cargo.toml`의 의존성을 확인한다.
   - 최근 컴파일 에러가 있다면 에러 내용을 분석한다.

2. **현재 단계 판별**:
   - 의존성 없음 + `std::fs` 미사용 → Stage 1 시작 전 또는 진행 중
   - `std::fs` 사용 중 → Stage 1~2
   - `toml`/`serde` 의존성 → Stage 3
   - `clap` 의존성 → Stage 4
   - `colored`/`terminal_size` → Stage 5

3. **점진적 힌트 제공** (3단계 깊이):

   사용자에게 인자가 있으면 (`$ARGUMENTS`) 해당 주제에 맞는 힌트를 제공한다.
   인자가 없으면 현재 코드에서 다음으로 진행해야 할 부분을 판별한다.

   **Level 1 — 개념 힌트**:
   - 어떤 모듈/타입/트레이트를 찾아봐야 하는지 알려준다
   - 예: "std::fs 모듈의 read_dir 함수를 찾아보세요"

   **Level 2 — 방향 힌트**:
   - 함수 시그니처, 반환 타입, 필요한 import를 알려준다
   - 예: "read_dir는 Result<ReadDir>를 반환합니다. ReadDir는 이터레이터예요."

   **Level 3 — 부분 코드 힌트**:
   - 핵심 부분의 뼈대만 보여준다 (완성된 코드는 절대 금지)
   - 예: "for entry in fs::read_dir(...)? { let entry = entry?; ... }"

4. **응답 형식**:

   ```
   ## 🔍 현재 상태
   Stage N 진행 중 — (현재 상태 요약)

   ## 💡 힌트
   (Level 1 힌트)

   더 자세한 힌트가 필요하면 `/hint`를 다시 입력하세요.
   ```

## 중요 규칙

- **완성된 코드 블록을 절대 제공하지 않는다.**
- 같은 주제에 대해 `/hint`를 반복하면 더 깊은 레벨의 힌트를 제공한다.
- 이전 대화의 힌트 레벨을 참고하여 다음 레벨로 진행한다.
- 사용자의 현재 코드에서 구체적으로 어디를 수정해야 하는지 위치를 알려줄 수 있다.
