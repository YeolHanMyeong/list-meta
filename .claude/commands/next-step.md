# /next-step 커맨드

현재 진행 상황을 분석하고, 다음에 해야 할 구체적인 작업을 안내한다.

## 실행 절차

1. **현재 상태 분석**:
   - `src/main.rs` 및 모든 `.rs` 소스 파일을 읽는다.
   - `Cargo.toml`의 의존성을 확인한다.
   - 현재 구현된 기능들을 파악한다.

2. **단계 및 진행도 판별**:
   - 현재 몇 단계인지 판별한다.
   - 해당 단계의 체크리스트에서 완료/미완료 항목을 파악한다.

3. **다음 작업 안내**:

   **현재 단계가 미완료인 경우:**
   - 체크리스트의 다음 미완료 항목을 안내한다.
   - 구현에 필요한 핵심 개념을 간단히 설명한다.
   - 관련 Rust API나 크레이트를 키워드로 안내한다.

   **현재 단계가 완료인 경우:**
   - 축하 메시지를 전한다.
   - 다음 단계를 소개한다.
   - 다음 단계에서 새로 배울 개념을 미리보기한다.
   - `Cargo.toml`에 추가해야 할 의존성을 안내한다 (해당되는 경우).

4. **응답 형식**:

   ```
   ## 📍 현재 위치
   Stage N: (단계 제목) — (진행 상태)

   ## ➡️ 다음 할 일
   (구체적인 작업 설명)

   ### 필요한 개념
   - (개념 1): (한 줄 설명)
   - (개념 2): (한 줄 설명)

   ### 시작 포인트
   (어디서부터 코드를 수정하면 좋은지 안내)

   💡 막히면 `/hint`를 입력하세요.
   ```

## 중요 규칙

- **코드를 대신 작성하지 않는다.**
- 구체적이지만 답을 주지 않는 안내를 한다.
- "무엇을 해야 하는지"는 명확히, "어떻게 하는지"는 스스로 찾도록 유도한다.
- 한 번에 하나의 작업만 안내하여 집중도를 높인다.
