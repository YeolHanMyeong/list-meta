# /review-step 커맨드

현재 학습 단계의 구현 상태를 체크리스트와 비교하여 리뷰한다.

## 실행 절차

1. **소스 코드 읽기**:
   - `src/main.rs` 및 `src/` 하위 모든 `.rs` 파일을 읽는다.
   - `Cargo.toml`의 의존성을 확인한다.

2. **현재 단계 판별**:
   - Cargo.toml 의존성과 소스 코드 분석으로 현재 단계를 판별한다.

3. **단계별 체크리스트 평가**:
   해당 단계의 참조 파일에 있는 체크리스트를 기준으로 평가한다.
   각 항목에 대해:
   - ✅ 완료 — 이 항목이 구현됨
   - ⚠️ 부분 완료 — 구현했지만 개선 필요
   - ❌ 미완료 — 아직 구현되지 않음

4. **코드 품질 평가**:
   - 에러 처리가 적절한가?
   - 관용적인 Rust 코드인가?
   - 불필요한 `unwrap()`, `.clone()` 등 안티패턴은 없는가?
   - 코드 구조가 깔끔한가?

5. **응답 형식**:

   ```
   ## 📋 Stage N 리뷰: (단계 제목)

   ### 체크리스트
   - ✅ (완료 항목)
   - ⚠️ (부분 완료 항목) — (개선 방향)
   - ❌ (미완료 항목)

   ### 잘한 점
   - (구체적 칭찬)

   ### 개선 제안
   1. (제안 + 힌트)
   2. (제안 + 힌트)

   ### 진행도: X/Y 완료
   (다음 할 일 한 줄 요약)
   ```

## 중요 규칙

- **코드를 대신 수정하지 않는다.**
- 개선 제안은 방향과 키워드만 제공한다.
- 체크리스트 항목이 모두 완료되면 다음 단계로 진행할 준비가 됐음을 알려준다.
- 현재 단계를 완벽히 끝내야 다음 단계로 가도록 안내한다.
